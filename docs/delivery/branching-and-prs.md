# Branching And PRs

## Rules

- Use PRs for cross-repo changes.
- Keep each repository PR focused on that repository's responsibility.
- Link related PRs from the control-center change note or issue.
- Merge order should follow source-of-truth dependencies.

## Suggested Merge Order

1. Platform contract or behavior PR.
2. SDK PR.
3. Mock platform and validation PRs.
4. Consumer PRs.
5. Control-center rollout or migration note.
