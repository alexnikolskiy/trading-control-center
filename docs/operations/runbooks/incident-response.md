# Runbook: Incident Response

## Steps

1. Declare incident scope and severity.
2. Identify affected repositories and owners.
3. Capture timeline, current symptoms, and correlation identifiers.
4. Stabilize the system.
5. Decide rollback, mitigation, or forward fix.
6. Record commands, deploys, and decisions.
7. Write postmortem in `docs/history/postmortems/`.
