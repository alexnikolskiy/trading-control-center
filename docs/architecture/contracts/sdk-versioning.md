# SDK Versioning

## Policy

- SDK versions must communicate compatibility with platform contract versions.
- Breaking consumer-facing changes require migration notes.
- New platform capabilities should include SDK examples when exposed to consumers.
- Pre-release SDK builds may be used for cross-repo validation before public release.

## Release Checklist

- Types updated.
- Client methods updated.
- Examples updated.
- Contract tests pass.
- Dependent consumers have been checked or pinned.
- Changelog entry exists.
